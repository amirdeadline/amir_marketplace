## Authoritative vs generated

| Path | Role |
|------|------|
| `.ai/state/*.json` | **Truth** вЂ” orchestrator updates via state tools |
| `.ai/project-goal.md` | **Truth** for goal вЂ” updated only through approved design/decision flow |
| `.ai/views/*.md` | **Generated** вЂ” never edit by hand except via regenerate |
| `.ai/agents/*` | **Workspace** вЂ” drafts, QA reports, logs, notes |

If hand-edited JSON conflicts with tools, run doctor and reconcile вЂ” do not fork truth in chat.

## Agent workspace rules

Each agent id (`core/naming-rules.md`) gets `.ai/agents/<agent-id>/`:

| Allowed | Purpose |
|---------|---------|
| `notes.md` | Working memory per `core/context-engineering.md` |
| `qa-report.md`, reviews | Filled templates from QA loop |
| `logs/` | Command output, scan results |
| `handoff.md` | Pause/resume snippets |

Rules:

- Workspace files are **evidence**, not decisions вЂ” promote outcomes to JSON
- Do not store secrets in workspace; use **PROBLEM** if task requires it
- Verifier workspaces (`qa-<task-id>`) live under `3-qa/` or `.ai/agents/qa-<task-id>/` per project template вЂ” be consistent within project
- Sub-agents use nested paths: `.ai/agents/dev-T001/sub-<name>/`

## Single-writer discipline

| Data | Writer |
|------|--------|
| `tasks.json`, `decisions.json`, `status.json`, `cost.json` | **Orchestrator only**, via `tools/` state commands |
| `activity.jsonl` | **Append-only** вЂ” any agent may append events; no deletes or rewrites |
| `views/` | Regenerated by tools вЂ” agents do not hand-edit |
| `project-goal.md` | Orchestrator after human-approved design/decision |

Workers and QA **propose** changes in artifacts; orchestrator merges to JSON.

Violations в†’ **PROBLEM**, log to activity.

## activity.jsonl

Append-only JSON lines:

```json
{"timestamp":"ISO-8601","agent_id":"dev-T001","task_id":"T001","event":"fix_cycle_end","details":{}}
```

Used for: budget events, drift checks, discovery batches, gate transitions, audit.

## Project isolation

- One `.ai/` tree **per project** repository or workspace root
- Agents must not read or write another project's `.ai/state/` unless explicit multi-repo spec in goal
- Host global config does not override project `decisions.json`
- `/project_create` initializes fresh state вЂ” no carryover of task ids or decisions

## Human visibility

- `/project_status`, `/project_watch` read views + state
- `/details` dumps specific workspace or view file on demand (`core/message-contract.md`)
- `/project_doctor` validates schema, orphan agents, stale views
